# Pain001: Automate ISO 20022-Compliant Payment File Creation

![Pain001 banner][banner]

## Overview

**Pain001** is an open-source Python Library that you can use to create
**ISO 20022-Compliant Payment Files** directly from your **CSV** or **SQLite**
Data Files.

- **Website:** <https://pain001.com>
- **Source code:** <https://github.com/sebastienrousseau/pain001>
- **Bug reports:** <https://github.com/sebastienrousseau/pain001/issues>

## 🚨 Breaking Changes

### Mandatory Data Validation

**This release introduces a breaking change:** Data validation is now **mandatory** across all data sources.

**What Changed:**
- Previously, data validation was optional and could be skipped
- Now, all data (CSV, SQLite, Python dict/list) is automatically validated
- Invalid data will raise a `ValueError` with a clear error message
- This ensures all payment files are ISO 20022-compliant

**Impact on Your Code:**

**Before v0.0.30:**
```python
from pain001 import main

# Invalid data could pass through silently
main('pain.001.001.03', 'template.xml', 'schema.xsd', 'data.csv')
```

**After v0.0.30:**
```python
from pain001 import main

try:
    main('pain.001.001.03', 'template.xml', 'schema.xsd', 'data.csv')
except ValueError as e:
    print(f"Data validation failed: {e}")
    # Fix your data according to the error message
```

**Validation Checks:**
- ✅ All required fields are present
- ✅ Data types are correct (strings, numbers, booleans)
- ✅ Boolean values are valid ('true'/'false' or 'yes'/'no')
- ✅ Field formats meet ISO 20022 standards

**Why This Change?**

Mandatory validation is the fundamental principle of Pain001 - ensuring data quality and ISO 20022 compliance. This change prevents silent failures and ensures banks won't reject your payment files due to data errors.

### Breaking Changes

**⚠️ IMPORTANT:** Data validation is now mandatory.

If your code was working with invalid data, you'll now receive explicit `ValueError` exceptions.

**Required Changes:**

1. **Add error handling** around all Pain001 calls:

```python
from pain001 import main

try:
    main(
        'pain.001.001.03',
        'template.xml',
        'schema.xsd',
        'data.csv'
    )
    print("✓ Payment file created successfully")
except ValueError as e:
    print(f"✗ Data validation failed: {e}")
    # Fix your data or alert the user
except Exception as e:
    print(f"✗ Error: {e}")
```

2. **Ensure your data is valid:**
   - All required fields must be present
   - Data types must be correct
   - Boolean values must be 'true'/'false' or 'yes'/'no'
   - Field formats must meet ISO 20022 standards

3. **Test with your data:**
   - Run your application with test data first
   - Catch any validation errors early
   - Fix data issues before production deployment

### Compatibility

- Python 3.9+ (unchanged)
- All valid data continues to work without modifications
- Invalid data will now raise `ValueError` instead of being silently processed
- Backward compatible for valid data with v0.0.29, v0.0.28, and v0.0.27
