## 🎉 ISO 20022 Format Expansion: pain.001.001.10 & pain.001.001.11 Support

## 📖 Usage Examples

### Using pain.001.001.10

```python
from pain001 import process_files

# Generate pain.001.001.10 XML from CSV data
process_files(
    csv_file_path="payments.csv",
    xml_message_type="pain.001.001.10",
    xsd_schema_file_path="pain001/templates/pain.001.001.10/pain.001.001.10.xsd",
    xml_template_file_path="output_payment.xml"
```

## 🔧 Technical Details

### XML Structure

Both v10 and v11 follow the standard ISO 20022 structure:

```xml
<?xml version="1.0"?>
<Document xmlns="urn:iso:std:iso:20022:tech:xsd:pain.001.001.10">
  <CstmrCdtTrfInitn>
    <GrpHdr>
      <MsgId>MSG001</MsgId>
      <CreDtTm>2023-03-10T15:30:47.000Z</CreDtTm>
      <NbOfTxs>2</NbOfTxs>
      <InitgPty><Nm>John Doe</Nm></InitgPty>
    </GrpHdr>
    <PmtInf>
      <!-- Payment Information -->
      <CdtTrfTxInf>
        <!-- Credit Transfer Transaction Information -->
      </CdtTrfTxInf>
    </PmtInf>
  </CstmrCdtTrfInitn>
</Document>
```

### Template Variables

Both templates support the standard Jinja2 placeholders:

**Group Header:**
- `{{id}}` - Message identification
- `{{date}}` - Creation date/time
- `{{nb_of_txs}}` - Number of transactions
- `{{initiator_name}}` - Initiating party name

**Payment Information:**
- `{{payment_id}}` - Payment identification
- `{{payment_method}}` - Payment method (TRF)
- `{{requested_execution_date}}` - Execution date
- `{{debtor_name}}` - Debtor name
- `{{debtor_account_IBAN}}` - Debtor IBAN
- `{{debtor_agent_BIC}}` - Debtor BIC
- `{{charge_bearer}}` - Charge bearer code

**Transaction Loop:**
- `{{tx.payment_id}}` - Transaction ID
- `{{tx.payment_amount}}` - Amount
- `{{tx.payment_currency}}` - Currency code
- `{{tx.creditor_agent_BIC}}` - Creditor BIC
- `{{tx.creditor_name}}` - Creditor name
- `{{tx.creditor_account_IBAN}}` - Creditor IBAN
- `{{tx.remittance_information}}` - Remittance info
