# Pain001 Release v0.0.47

**Release Date:** 2026-01-18
**Tag:** v0.0.47
**Python:** 3.9+
**Status:** Stable

## Executive Summary

v0.0.47 focuses on **Cloud-Native Architecture** and **Security Hardening**. It introduces full I/O decoupling for serverless environments (AWS Lambda/Azure Functions), O(1) memory streaming data loaders for processing large datasets, and critical security mitigations for Log Injection (CWE-117), SQL Injection (CWE-89), and Path Traversal.

## New Features

### ☁️ Serverless I/O Decoupling (PR #152)

Generate XML directly to memory strings without file system dependencies:

- **`generate_xml_string()`** - Returns XML as a string instead of writing to a file.
- **Why?** - Eliminates disk writes, enabling seamless integration with:
  - AWS Lambda / Azure Functions / Google Cloud Run
  - API Gateway responses
  - Containerized environments (Docker/K8s)
- **Streaming Support:** Optimized for Low-Memory Environments.

### ⚡ O(1) Streaming Data Loaders (PR #152)

Process massive payment files with constant memory usage:

- **`load_csv_data_streaming()`** - Configurable chunk processing (default: 1000 rows).
- **`load_db_data_streaming()`** - Generator-based SQLite query execution.
- **Impact:** ~90% memory reduction for datasets > 10,000 transactions.

## Security Hardening 🛡️

### Log Injection Protection (CWE-117)
- **Fix:** Implemented strict sanitization for all logged data.
- **Mechanism:** Strips `\n` and `\r` characters from user inputs before logging.
- **Coverage:** Applied to CSV streaming loaders and core logging schema.

### SQL Injection Hardening (CWE-89)
- **Fix:** Replaced table name transformation with strict regex validation (`^[a-zA-Z][a-zA-Z0-9_]*$`).
- **Mechanism:** Rejects invalid table identifiers instead of attempting sanitization.
- **Coverage:** Both standard and streaming SQLite loaders.

### Path Traversal Mitigation
- **Fix:** Enhanced path validation logic across all XML templates (v03-v11).
- **Mechanism:** Pre-validation with allowlist checking before `Path()` operations.
- **Result:** Resolved 21 HIGH-severity CodeQL alerts.

## Bug Fixes

- **XML String Normalization:** Fixed byte-for-byte regression ensuring `xml_to_string()` output matches legacy file-based writer.
- **CI/CD Template Loading:** Fixed `FileNotFoundError` in GitHub Actions by using package-relative path resolution for Jinja2 templates.
- **Package Structure:** Added missing `__init__.py` to `pain001/schemas/` directory to fix distribution issues.
- **CLI Complexity:** Refactored `main()` to Grade A complexity.

## Performance

- **Memory:** Constant O(1) memory usage for streaming loaders.
- **Speed:** XML generation < 500ms per 1000 transactions.
- **Coverage:** 93.32% (High confidence).
