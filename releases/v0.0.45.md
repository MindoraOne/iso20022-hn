# Pain001 Release v0.0.45

**Release Date:** 2026-01-13  
**Tag:** v0.0.45  
**Python:** 3.9+  
**Status:** Stable

## Executive Summary

This release introduces two major enhancements for validation workflows and observability: a CLI dry-run mode for pre-flight validation without XML generation, and enterprise-grade structured logging with standardized events and PII protection.

## New Features

### CLI Dry-Run Mode ([#81](https://github.com/sebastienrousseau/pain001/issues/81))

Validates XML template, XSD schema, and payment data without generating XML files, enabling CI/CD pre-flight checks and validation workflows.

**Key Features:**
- `--dry-run` / `--validate-only` flag for validation-only mode
- Returns exit code 0 on success, 1 on validation failure
- Validates template structure, XSD compliance, and data integrity
- Supports all input sources (CSV, SQLite, Python list/dict)
- Available in CLI and programmatic API

**Example:**
```bash
python3 -m pain001 \
  -t pain.001.001.03 \
  -m template.xml \
  -s schema.xsd \
  -d data.csv \
  --dry-run
```

**Use Cases:**
- Pre-commit hooks for data validation
- CI/CD pipeline integration
- Data quality checks before batch processing
- Template and schema validation during development

### Structured Logging Normalization ([#102](https://github.com/sebastienrousseau/pain001/issues/102))

Standardized event names and fields across CLI and library for enterprise-grade observability and log aggregation.

**Key Features:**
- 17 standardized event types (process lifecycle, validation, data loading, XML generation, XSD validation, namespace registration)
- 20 standardized field names (event, timestamp, message_type, iso_version, record_count, duration_ms, etc.)
- JSON-formatted log entries for machine parsing
- PII masking utilities for IBAN, BIC, names, and amounts
- Helper functions for common logging patterns
- Comprehensive documentation and test coverage

**Example:**
```python
from pain001.logging_schema import (
    Events,
    Fields,
    log_process_start,
    log_process_success,
    mask_sensitive_data,
)

# Start process with timing
start_time = log_process_start(
    logger,
    message_type="pain.001.001.03",
    data_source_type="csv",
)

# ... perform operations ...

# Log success with duration
log_process_success(
    logger,
    start_time=start_time,
    message_type="pain.001.001.03",
    record_count=100,
)

# Mask sensitive data
masked_iban = mask_sensitive_data(
    "AT611904300234573201", 
    data_type="iban"
)  # Returns: AT61****3201
```

**Log Aggregation Ready:**
- Elasticsearch compatible
- Splunk ready
- AWS CloudWatch integration
- Datadog compatible

**PII Protection:**
- IBAN masking: Shows first 4 and last 4 chars only
- BIC masking: Replaces middle characters
- Amount redaction in INFO logs
- Full data only at DEBUG level with explicit consent

## Migration Guide

No migration needed. Install and use as normal.

**To use dry-run mode:**
```bash
# Add --dry-run flag to existing commands
python3 -m pain001 -t pain.001.001.03 -m template.xml -s schema.xsd -d data.csv --dry-run
```

**To use structured logging:**
```python
# Import and use standardized logging helpers
from pain001.logging_schema import log_process_start, log_process_success

# Existing code continues to work unchanged
# Structured logging is used internally by default
```
