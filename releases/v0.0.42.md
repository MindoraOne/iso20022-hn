# Pain001 v0.0.42 Release Notes

## Release Date
January 11, 2026

## Overview
Version 0.0.42 fixes duplicate workflow executions that occurred during release deployments.

## Fixed

### Workflow Execution
- **Duplicate Runs**: Removed tag trigger from docs workflow
- **Race Conditions**: Docs now only triggered via workflow_call from release workflow
- **Resource Usage**: Prevents parallel documentation builds

## Workflow Triggers

### Documentation Workflow
Now only triggers on:
- Push to main with doc/version file changes
- Manual dispatch (workflow_dispatch)
- Called from release workflow (workflow_call)

### Release Flow
Clean execution path on tag push:
1. Release workflow detects tag
2. Builds and uploads package to PyPI
3. Creates GitHub release
4. Calls docs workflow (single execution)
5. Deploys documentation with correct version

## Benefits
- Single workflow execution per release
- No duplicate deployments
- Cleaner workflow logs
- Reduced GitHub Actions minutes usage
