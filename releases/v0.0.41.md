# Pain001 v0.0.41 Release Notes

## Release Date
January 11, 2026

## Overview
Version 0.0.41 improves the release workflow to prevent version conflicts and ensure GitHub releases are only created after successful PyPI uploads.

## Changed

### Release Workflow Improvements
- **GitHub Release Creation**: Now happens only after successful PyPI upload
- **Duplicate Handling**: Added `--skip-existing` flag to handle cases where version already exists on PyPI
- **Release Notes**: Automated GitHub release creation now includes release notes from `releases/` directory
- **Artifacts**: Distribution files are attached to GitHub releases

## Fixed

### Release Process Issues
- Fixed orphaned GitHub releases when PyPI upload fails
- Prevented workflow failures when version already exists on PyPI
- Ensured version consistency between PyPI and GitHub releases

## Workflow Behavior

### New Release Flow
1. Build package distributions (wheel and sdist)
2. Upload to PyPI with `--skip-existing` flag
3. Create GitHub release only if PyPI upload succeeds
4. Attach distribution files to GitHub release
5. Include release notes from version-specific markdown file

### Error Handling
- If version exists on PyPI: Skip upload, continue to GitHub release
- If PyPI upload fails: Stop workflow, no GitHub release created
- If GitHub release exists: Update with new artifacts
